# ABSTRACT

This study presents the development of a Domain-Specific Language (DSL) for task automation aimed at simplifying the process of automating repetitive tasks in computing environments.

Existing automation methods rely on general-purpose programming languages such as Python and Bash, which are often complex and difficult for non-technical users to understand and apply. This creates challenges in usability, maintenance, and standardization of automation processes.

The aim of this research is to design and implement a simple and user-friendly DSL that allows users to automate tasks using clear and structured commands.

The methodology adopted is the Object-Oriented Analysis and Design Methodology (OOADM), where the system is developed using modular components such as the parser, interpreter, and scheduler to process and execute automation commands.

The proposed system provides a simplified way of performing tasks such as file management, scheduling, and report generation without requiring advanced programming knowledge. It improves efficiency, reduces dependence on technical expertise, and enhances task execution consistency.

The developed DSL offers a more accessible and efficient approach to task automation and can be used by both technical and non-technical users to improve productivity and reduce manual effort.

#### Language-Oriented Programming Theory

Language-oriented programming focuses on building software systems using specialized languages designed for specific problem domains.

#### Abstraction Theory in Programming

Abstraction theory explains how complex systems can be simplified by hiding implementation details.

#### Formal Language Theory

Formal language theory defines how programming languages are structured using syntax, grammar, and semantics.

#### Human-Computer Interaction Theory

HCI theory studies how users interact with computer systems and how systems can be designed for better usability.

#### Software Reusability and Modularity Theory
